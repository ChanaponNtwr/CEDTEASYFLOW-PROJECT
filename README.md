# CE68-37  
## แพลตฟอร์มเพื่อศึกษาการเขียนโปรแกรมด้วยการเขียนผังงาน  
**CEDEASYFLOW-PROJECT**

ระบบสำหรับการเรียนรู้การเขียนโปรแกรมผ่านการสร้างผังงาน ประกอบด้วย 2 ส่วนหลัก ได้แก่

- **Frontend (`easyflow`)** สำหรับผู้ใช้งานและผู้เรียน
- **Backend (`backend`)** สำหรับจัดการ API, Authentication, และฐานข้อมูล

---

## คุณสมบัติของระบบ

- รองรับการใช้งานร่วมกับ **PostgreSQL**
- ใช้ **Next.js** สำหรับ Frontend
- ใช้ **Node.js** สำหรับ Backend
- รองรับ **Google OAuth / NextAuth**
- รองรับการส่งอีเมลผ่าน **SMTP**
- ใช้ **Prisma** ในการจัดการฐานข้อมูล

---

## การติดตั้งโปรแกรม

### วิธีที่ 1: ติดตั้งจาก Git (Clone / Pull)

วิธีนี้เหมาะสำหรับผู้พัฒนาหรือผู้ดูแลระบบที่ต้องการติดตั้งจากซอร์สโค้ดโดยตรง

#### 1) ติดตั้งโปรแกรมพื้นฐาน

ตรวจสอบให้มีโปรแกรมดังต่อไปนี้ก่อนเริ่มใช้งาน

* Git
* Node.js 20 ขึ้นไป
* npm
* PostgreSQL 16
* หรือ Docker / Docker Compose สำหรับการใช้งานแบบ container

#### 2) Clone โปรเจกต์จาก GitHub

```bash
git clone https://github.com/ChanaponNtwr/CEDTEASYFLOW-PROJECT.git
cd CEDTEASYFLOW-PROJECT
```

#### 3) ติดตั้ง Backend

```bash
cd backend
npm install
```

#### 4) ติดตั้ง Frontend

```bash
cd ../easyflow
npm install
```

#### 5) สร้างไฟล์ `.env` และกำหนดค่าให้ครบ

ก่อนเริ่มใช้งาน ควรตั้งค่า environment variables ให้ถูกต้องทั้งในฝั่ง Backend และ Frontend

### ตัวแปรที่ต้องตั้งค่า

| ส่วนที่ใช้ | ตัวแปรที่ควรกำหนด                                                                                                                                                                | คำอธิบาย                                                         |
| ---------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ---------------------------------------------------------------- |
| Backend    | `DATABASE_URL`<br>`NEXTAUTH_SECRET`<br>`GOOGLE_CLIENT_ID`<br>`GOOGLE_CLIENT_SECRET`<br>`NEXTAUTH_URL`<br>`SMTP_HOST`<br>`SMTP_PORT`<br>`SMTP_USER`<br>`SMTP_PASS`<br>`SMTP_FROM` | ใช้สำหรับเชื่อมต่อ PostgreSQL, ยืนยันตัวตน, และส่งอีเมล          |
| Frontend   | `DATABASE_URL`<br>`NEXTAUTH_SECRET`<br>`GOOGLE_CLIENT_ID`<br>`GOOGLE_CLIENT_SECRET`<br>`NEXTAUTH_URL`<br>`BASE_API`                                                              | ใช้สำหรับเชื่อมต่อฐานข้อมูล, ยืนยันตัวตน, และเรียกใช้งาน Backend |
| Prisma     | `schema.prisma` / migration                                                                                                                                                      | ใช้ตรวจสอบและกำหนดโครงสร้างฐานข้อมูล                             |

> ถ้าใช้งานผ่าน Docker ควรตรวจสอบว่าไฟล์ `Dockerfile` ของทั้งสองส่วนถูกตั้งค่าเรียบร้อย โดย **Frontend ใช้พอร์ต 3000** และ **Backend ใช้พอร์ต 8080**

#### 6) สร้าง Prisma Client และทำ Migration

```bash
cd ../backend
npx prisma generate
npx prisma migrate dev --name init
```

> ตาราง `Role` ต้องสอดคล้องกับโครงสร้างที่ระบบใช้งานจริง

#### 8) รันระบบ

เปิด Terminal แยกกัน 2 หน้าต่าง แล้วสั่งรันแต่ละส่วนดังนี้

**Backend**

```bash

```

**Frontend**

```bash
cd easyflow

```

## การกำหนดค่าระบบ

### คำอธิบายตัวแปรสำคัญ

* **DATABASE_URL**
  ใช้ระบุที่อยู่ของฐานข้อมูล PostgreSQL

* **NEXTAUTH_SECRET**
  ใช้สำหรับเข้ารหัสและยืนยันตัวตนของ NextAuth

* **GOOGLE_CLIENT_ID / GOOGLE_CLIENT_SECRET**
  ใช้สำหรับการเข้าสู่ระบบด้วย Google OAuth

* **NEXTAUTH_URL**
  ใช้กำหนด URL หลักของระบบยืนยันตัวตน เพื่อให้ callback และ session ทำงานได้ถูกต้อง

* **SMTP_HOST / SMTP_PORT / SMTP_USER / SMTP_PASS / SMTP_FROM**
  ใช้สำหรับตั้งค่าการส่งอีเมล เช่น อีเมลยืนยันตัวตนหรือการแจ้งเตือน

* **BASE_API**
  ใช้ระบุ URL ของ Backend ที่ Frontend ต้องเรียกใช้งาน API

* **Prisma (`schema.prisma` / migration)**
  ใช้กำหนดโครงสร้างฐานข้อมูล และควรตรวจสอบให้ตรงกับฐานข้อมูลจริงก่อน `generate` หรือ `migrate`

## การตรวจสอบหลังติดตั้ง

หลังจากติดตั้งเสร็จ ควรตรวจสอบการทำงานดังนี้

* เปิด Frontend ผ่านเว็บเบราว์เซอร์และตรวจสอบว่าหน้าแรกโหลดได้
* ทดสอบการเรียก API จาก Backend
* ตรวจสอบว่าฐานข้อมูล PostgreSQL สามารถอ่าน/เขียนข้อมูลได้
* ทดสอบการส่งอีเมลและระบบยืนยันตัวตน หากมีการใช้งาน SMTP และ NextAuth

---

## ปัญหาที่พบบ่อยและแนวทางแก้ไข

### 1) ติดตั้ง dependency ไม่สำเร็จ

* ตรวจสอบเวอร์ชันของ Node.js และ npm ว่าเป็นเวอร์ชันที่ระบบรองรับ

### 2) เชื่อมต่อฐานข้อมูลไม่ได้

* ตรวจสอบค่า `DATABASE_URL`
* ตรวจสอบชื่อผู้ใช้ รหัสผ่าน และพอร์ตของ PostgreSQL

### 3) รัน Prisma ไม่ผ่าน

* ตรวจสอบว่าได้ติดตั้ง dependency แล้ว
* สั่ง `npx prisma generate` ก่อน `migrate`

### 4) Frontend ติดต่อ Backend ไม่ได้

* ตรวจสอบค่า `BASE_API`
* ตรวจสอบพอร์ตของ Backend ว่าเปิดใช้งานอยู่

### 5) ส่งอีเมลไม่ได้

* ตรวจสอบค่า SMTP
* ตรวจสอบรหัสผ่านแอปพลิเคชันของบัญชีอีเมล
* ตรวจสอบการอนุญาตการส่งอีเมลจากผู้ให้บริการ
